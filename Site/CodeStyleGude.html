CSS Coding Style Rules

Class Names for styling

ID's are not to be used for any styling | unless absolutly nessicary. This being said ALWAYS use classes UNLESS all else fails. ID's are a absolute LAST RESORT and you should review your CSS Code before you even attempt to use an ID

  • All global element classes should begin with a lowercase g. Examples: .gHeader | .gWrap | .gFooter
  • .fullWrap
  • .full
  • .half
  • .quarter
  • .mOnly is specifed for elements that are ONLY designed to appear on a mobile device
  • .nonM is specifed for elements that are NOT designed to appear on a mobile device

SCSS document name declaration should always be stated at the top of the SCSS file. First comment line (Code Line 1) will start with regular SCSS comment syntax's // and is followed by 50 equal signs ( = ) and then two // and then a hard return. Second comment line (Code Line 2) will have 71 spaces followed by two // to ensure the diagonal code line proceeds down to the next code line. Third comment line (Code Line 3) should have 5 spaces followed by two // followed by another space followed by the SCSS document name .scss. For example // pagename.scss. The page name should be followed by a hard return. Forth comment line (Code Line 4) will have 69 spaces followed by two // followed by 5 hyphens ( - ) next a square open bracket ( [ ) followed by a space then the page name in all caps and next a hard return. Fifth comment line (Code Line 5) will start with two // followed by 46 equal sign's ( = ) then two // to complete the document name declaration. If for some reason you need to further explain the page in detail use the muliple line comment rule after 4 hard returns after the document name declaration.

Multiple line comments

Multiple line comments start with the SCSS multiple line comment syntax's ( /* ). After the multiple line comment is declared a hard return is necessary to bring you down to comment line 2. On comment lines in-between the SCSS multiple line comment characters all text should be indented with a tab (or 4 spaces if your tabs are not set to 4 spaces). On line comment lines 2-4 of the multiple line comment there needs to be a title to declare what you are commenting about. this title section should be as followed, when tabbed in use 50 hyphens ( - ) to start the title section. Then add a hard return and tab in if not already tabbed to the correctly specified multiple line comment indention. Next add a star character ( * ) followed by a space and next the title in all caps. after the title hard return down to the next line, tab if not done so already and follow tab by 50 hyphens ( - ). This will end the title section of the multiple line comment. Each comment line within a multiple line comment should have a hard return before and after. next the mutiple line comment is closed with regular SCSS syntacts ( */ ). This completes the mutiple line comment.

/*

	-------------------------------------------------- 
	* Title before Number Syentacts
	-------------------------------------------------- 

	1  | This page is created for all global design elements. ( Header | Footer | Schedule )
	
	2  | I think that this is a good example of the number syntacts.
	
	3  | This is a thrid option in the list here.

*/

Multiple numbered line comments

Multiple numbered line comments start with the SCSS multiple line comment syntax's ( /* ). After the multiple line comment is declared a hard return is necessary to bring you down to comment line 2. On comment lines in-between the SCSS multiple line comment characters all text should be indented with a tab (or 4 spaces if your tabs are not set to 4 spaces). On line comment lines 2-4 of the multiple line comment there needs to be a title to declare what you are commenting about. this title section should be as followed, when tabbed in use 50 hyphens ( - ) to start the title section. Then add a hard return and tab in if not already tabbed to the correctly specified multiple line comment indention. Next add a star character ( * ) followed by a space and next the title in all caps. after the title hard return down to the next line, tab if not done so already and follow tab by 50 hyphens ( - ). This will end the title section of the multiple line comment. Each numbered comment line within a multiple numbered line comment should have a hard return before and after. Also each line should start with a number then followed by 2 spaces, then a vertical bar ( | ), a space and then the comment.

/*
	-------------------------------------------------- 
	* Title before Number Syentacts
	-------------------------------------------------- 

	1  | This page is created for all global design elements. ( Header | Footer | Schedule )
	
	2  | I think that this is a good example of the number syntacts.
	
	3  | This is a thrid option in the list here.

*/
